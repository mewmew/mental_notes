Interface

..cantor.dust.. is built around an extensible plugin interface.
New visualization techniques are easily created and added as needed.

The user navigates through the file/memory dump/packet/etc using multiple navigation panes (left),
while analyzing the data with the selected visualization technique (right).

With a little practice, the analyst discovers that
some techniques are best suited for analyzing the underlying data type,
some are proficient at drawing out anomalies,
some can easily illustrate data interrelationships,
and others can instantly extract areas of interest to the reverser.


Depending on the application and the information being sought,
the user can focus on a single technique, or every technique at once.
Only a fraction of the available visualization methods are pictured.


User interaction is a key component of ..cantor.dust..
Each visualization technique is dynamic,
providing instant feedback as the user explores the data.
Visualizations can be quickly adjusted by the investigator,
in order to emphasize the specific features the reverser is seeking.

The skilled user can visually sift through gigabytes of data
in a matter of minutes,
easily identifying underlying data types based on their visual signature,
while isolating areas of interest and data anomalies.

..cantor.dust.. serves as a visual extension to the modern hex editor.
It allows a user to immediately see data types and relationships that would
take days of arduous work to understand at a binary level.
Yet at the same time the tool is data agnostic -
it makes no assumptions about the underlying data type being analyzed,
thus freeing it from the countless risks and pitfalls associated
with file headers, magic numbers, and obfuscation techniques.
